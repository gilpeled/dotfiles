# Homework

Short reference cards for the tools that came with this dotfiles setup. One file per tool, each ~5 minutes to read. Audience is future-Gil — an experienced engineer who writes Python/Java/Swift but is new to most CLI power-user tooling. The goal is to get from "installed" to "muscle memory" without slogging through upstream docs.

Each guide is self-contained. Read whichever one is in front of you. The order below is just the suggested learning path.

## Suggested reading order

1. **Start with `01-stow.md`** — explains how this whole repo works. Without it, the other configs feel magical.
2. **Daily-driver shell upgrades** — these change muscle memory most. Read in order or grab the one you're curious about:
   - `02-oh-my-zsh.md`
   - `03-powerlevel10k.md`
   - `04-eza.md`
   - `05-bat.md`
   - `06-fd.md`
   - `07-fzf.md`
   - `08-zoxide.md`
   - `09-ripgrep.md`
   - `10-git-delta.md`
   - `13-tldr.md`
3. **High-leverage power tools** — bigger learning curve, bigger payoff:
   - `16-tmux.md`
   - `17-aerospace.md`
   - `18-uv.md`
4. **Occasional / contextual** — read when you actually need them:
   - `11-glow.md`
   - `12-lazydocker.md`
   - `14-btop.md`
   - `15-dua-cli.md`
   - `19-mas.md`
   - `20-rtk.md`

## How to use

Open one file. Skim "Why you'll like it" and "The 80% you'll actually use." Then run the commands in "Try it now" in a real terminal. That's where the muscle memory starts.

When something surprises you, check "Gotchas / tips" before going to the upstream docs.
