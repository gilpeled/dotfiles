# fd

**`find`, but with a syntax that makes sense.**

`fd` searches for files and directories by name. It does what 95% of `find` invocations do — `find . -name "*foo*"` — but with a syntax humans can remember: `fd foo`. It's also faster (parallel by default), respects `.gitignore`, and has color output.

## Why you'll like it

- The pattern is a regex on the *filename*, not on the absolute path. `fd config` finds anything named like `config`, anywhere under cwd.
- Respects `.gitignore` and `.fdignore` automatically — no more results polluted by `node_modules/`.
- Colored, sensible defaults. `find`'s output is a wall of `./`-prefixed paths.
- Common operations are flags, not predicate trees: `-e py` for extension, `-t f` for files, `-H` for hidden.
- Works as input to `xargs` or `--exec` directly.

## Install

If installed via the dotfiles Brewfile, no action. Otherwise: `brew install fd`.

## The 80% you'll actually use

| Command | What it does |
| --- | --- |
| `fd foo` | Find anything matching `foo` under cwd. Case-insensitive if pattern is lowercase. |
| `fd -e py` | All `.py` files under cwd. `-e` is "extension." |
| `fd -e py -e pyi` | Multiple extensions. |
| `fd -t f foo` / `fd -t d foo` | Restrict to files / directories. |
| `fd -H foo` | Include hidden files (`.git`, `.env`, etc.). |
| `fd -I foo` | Don't respect `.gitignore`. |
| `fd foo /some/path` | Search a specific root, not cwd. |
| `fd -e py -x ruff check {}` | For each match, run a command. `{}` = the path. |
| `fd -e py -X wc -l` | Same but pass all matches as one big arg list (`xargs` style). |

## Gotchas / tips

- The pattern is a **regex**, not a glob. `fd 'foo.*\.py'` works; `fd '*.py'` does not. For globs, use `--glob` (or just lean on `-e`).
- Default is case-insensitive only when pattern is all lowercase. Mixed case → case-sensitive. (Same convention as `ripgrep`.)
- It will not search files matched by `.gitignore` — including `node_modules/`, `.venv/`, `.build/`. That's usually right; `-I` to override.
- `fd` searches the *filename component* by default. To match against the full path, use `-p` (`--full-path`).

## Try it now

```bash
# All Swift files in your iOS project
fd -e swift ~/gitRepos/SimplyPiano | head

# Anything containing the word "Piano" in its filename
fd Piano ~/gitRepos/SimplyPiano

# All Python files in your dotfiles repo, including dotfiles
fd -H -e py ~/dotfiles

# Find every README.md and run wc -l on each
fd -t f 'README\.md' ~/gitRepos -X wc -l

# Just directories named 'Pods', anywhere
fd -t d '^Pods$' ~/gitRepos
```
